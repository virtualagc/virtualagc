## What This Is

A port of `SDFPKG.ASM` (IBM System/360 Basic Assembly Language) to C and Python 3. SDFPKG is the HAL/S Space Shuttle compiler's "Symbol Data File" (SDF) access package ŌĆö a virtual-memory paging library for reading/writing structured compiler metadata files. All source files in this directory were generated by Claude (claude-sonnet-4-6) under direction from RSB; no file has been hand-edited by humans.

The C port targets integration into HALSFC (C-based HAL/S compiler). The Python port (`sdfpkg.py`) targets integration into `HAL_S_FC.py`.

## Architecture

### Flat File Format (replaces z/OS PDS)

```
4 bytes   magic 0x53444600 ("SDF\0")
4 bytes   member count N (big-endian u32)
N ├Ś 20    name(8 ASCII space-padded) + page_count(4 BE u32) + pg0_offset(8 BE u64)
N ├Ś pages member data: page_count ├Ś 1680 bytes per member
```

### Virtual Pointer Encoding

`sdf_vptr_t` is a 32-bit big-endian value: high 16 bits = page number, low 16 bits = byte offset within a 1680-byte page. Null pointer = 0. Macros: `SDF_VPTR_PAGE`, `SDF_VPTR_OFFSET`, `SDF_VPTR_MAKE`.

### Python Module Structure (`sdfpkg.py`)

Mirrors the C API exactly. Key classes:
- `SdfContext` ŌĆö main handle; `open()` / `close()` / context manager
- `BlockResult`, `SymbolResult`, `StmtResult` ŌĆö dataclass return types
- `SdfWriter` ŌĆö write path; `add_member()` ŌåÆ `add_block()` ŌåÆ `add_symbol()` ŌåÆ `commit()` ŌåÆ `write(path)`
- `WBlock`, `WSymbol` ŌĆö dataclasses for write-path inputs

### Write Path Design

The write path uses a bump allocator starting at page 1 (page 0 = PAGEZERO + DROOTCEL). Structures never straddle page boundaries. The block name binary search tree uses balanced median-split over the sorted block array. Symbols are sorted block-first, then alphabetically within each block; `fsymb_no..lsymb_no` ranges on each BLKTCELL are contiguous.

## Data Model

### Symbol Classes (`sym_class`)
From `##DRIVER.xpl` VAR_CLASS..TPL_FUNC_CLASS:
`SCLASS_VARIABLE=1`, `SCLASS_LABEL=2` (NAME vars and equate-externals), `SCLASS_FUNC=3`, `SCLASS_REPL_ARG=5`, `SCLASS_REPLACE=6`, `SCLASS_TEMPLATE=7`, `SCLASS_TPL_LAB=8`, `SCLASS_TPL_FUNC=9`

Equate-external symbols use `SCLASS_LABEL=2` with `STYPE_IORS=8` (detected by SDFPKG.ASM via `CLI CLASS,2 / CLI TYPE,8`).

### Symbol Types (`sym_type`)
From `##DRIVER.xpl` BIT_TYPE..EQUATE_LABEL:
`STYPE_BIT=1`, `STYPE_CHARACTER=2`, `STYPE_MATRIX=3`, `STYPE_VECTOR=4`, `STYPE_SCALAR=5`, `STYPE_INTEGER=6`, `STYPE_IORS=8`, `STYPE_EVENT=9`, `STYPE_STRUCTURE=10`, `STYPE_ANY=11`

Label types (hex): `STYPE_TEMPL_NAME=0x3E`, `STYPE_STMT_LABEL=0x42`, `STYPE_UNSPEC_LABEL=0x43`, `STYPE_IND_CALL_LAB=0x45`, `STYPE_CALLED_LABEL=0x46`, `STYPE_PROC_LABEL=0x47`, `STYPE_TASK_LABEL=0x48`, `STYPE_PROG_LABEL=0x49`, `STYPE_COMPOOL_LABEL=0x4A`, `STYPE_EQUATE_LABEL=0x4B`

HAL/S BOOLEAN is `STYPE_BIT` with `rows=1` (no separate BOOLEAN type).

### Block Classes (`blk_class`)
From `##DRIVER.xpl` PROC_MODE..INLINE_MODE:
`BCLASS_PROCEDURE=1` (also STRUCTURE templates), `BCLASS_FUNCTION=2`, `BCLASS_COMPOOL=3`, `BCLASS_PROGRAM=4`, `BCLASS_TASK=5`, `BCLASS_UPDATE=6`, `BCLASS_INLINE=7`

### Special Structures
- **STRUCTURE template**: `blk_class=1` (BCLASS_PROCEDURE); header symbol has `sym_class=7` (SCLASS_TEMPLATE), `sym_type=10` (STYPE_STRUCTURE), `rows=own symb_no`
- **STRUCTURE COPY**: SDC carries a 2-byte STRCDATA block at `struct_of` offset; `copy_blk_no` = source block number
- **ARRAY variable**: SDC has ARRADATA at `array_off` offset (8 bytes: `arraynum(H) + range1(H) + range2(H) + range3(H)`)
- **SDC fixed layout**: 24 bytes fixed, then name continuation (bytes 24ŌĆō47), then optional ARRADATA, then optional STRCDATA

## Known Bugs Fixed (History)

The `chatSummary.md` file documents the full history of bugs fixed during this project. Key non-obvious invariants:
- Symbol binary search uses 8-char compare only for lo/hi adjustment; `chk_match` is called only in linear search after an 8-char match.
- `name_cont` in SDC is at byte offset 24 (not 23).
- Extent cell `fst_off`/`lst_off` are absolute byte offsets from the page start.
- Symbol sort order: block-first, then alphabetically within block (not a global alphabetical sort).
- `chk_match` class logic: classes 1ŌĆō3 accepted unconditionally; classes > 3 only when `flag1 & 0x03` has exactly one bit set.
