Source Code

These source-code files derive from a printout of Luminary 116 (the Apollo 12 Lunar Module guidance computer program), from the personal library of original AGC developer Don Eyles, digitally photographed at archive.org, financially sponsored by Ron Burkey, and transcribed to source code by a team of volunteers. This colorized, syntax-highlighted form was created by assembling that transcribed source code. Note that the full page images are available on the Virtual AGC project page at archive.org, while reduced-size images are presented at the VirtualAGC project website. Report or fix any transcription errors at the Virtual AGC project code repository.

Notations on the program listing read, in part:

	GAP:  ASSEMBLE REVISION 116 OF AGC PROGRAM LUMINARY BY NASA 2021112-071
	19:09 AUG. 11,1969
Note that the date is the date of the printout, not the date of the program revision.

003543,000025:                                                                                                  #  CONVENTIONS AND NOTATIONS UTILIZED FOR ERASABLE ASSIGNMENTS.
003544,000026: 
003545,000027:                                                                                                  #           EQUALS IS USED IN TWO WAYS.  IT IS OFTEN USED TO CHAIN A GROUP
003546,000028:                                                                                                  #                  OF ASSIGNMENTS SO THAT THE GROUP MAY BE MOVED WITH THE
003547,000029:                                                                                                  #                  CHANGING OF ONLY ONE CARD.  EXAMPLE.
003548,000030: 
003549,000031:                                                                                                  #                                          X        EQUALS START
003550,000032:                                                                                                  #                                          Y        EQUALS X        +SIZE.X
003551,000033:                                                                                                  #                                          Z        EQUALS Y        +SIZE.Y
003552,000034: 
003553,000035:                                                                                                  #                  (X, Y, AND Z ARE CONSECUTIVE AND BEGIN AT START.       )
003554,000036:                                                                                                  #                  (SIZE.X AND SIZE.Y ARE THE RESPECTIVE SIZES OF X AND Y,
003555,000037:                                                                                                  #                   USUALLY NUMERIC,  IE. 1, 2, 6, 18D ETC.               )
003556,000038: 
003557,000039: 
003558,000040:                                                                                                  #           EQUALS OFTEN IMPLIES THE SHARING OF REGISTERS (DIFFERENT NAMES
003559,000041:                                                                                                  #                  AND DIFFERENT DATA).  EXAMPLE.
003560,000042: 
003561,000043:                                                                                                  #                                          X        EQUALS Y
003562,000044: 
003563,000045: 
003564,000046:                                                                                                  #           = MEANS THAT MULTIPLE NAMES HAVE BEEN GIVEN TO THE SAME DATA.
003565,000047:                                                                                                  #                  (THIS IS LOGICAL EQUIVALENCE, NOT SHARING)  EXAMPLE.
003566,000048: 
003567,000049:                                                                                                  #                                          X        =      Y
003568,000050: 
003569,000051: 
003570,000052:                                                                                                  #           THE SIZE AND UTILIZATION OF AN ERASABLE ARE OFTEN INCLUDED IN
003571,000053:                                                                                                  #           THE COMMENTS IN THE FOLLOWING FORM.  M(SIZE)N.
003572,000054: 
003573,000055:                                                                                                  #                  M REFERS TO THE MOBILITY OF THE ASSIGNMENT.
003574,000056:                                                                                                  #                       B   MEANS THAT THE SYMBOL IS REFERENCED BY BASIC
003575,000057:                                                                                                  #                           INSTRUCTIONS AND THUS IS E-BANK SENSITIVE.
003576,000058:                                                                                                  #                       I   MEANS THAT THE SYMBOL IS REFERENCED ONLY BY
003577,000059:                                                                                                  #                           INTERPRETIVE INSTRUCTIONS, AND IS THUS E-BANK
003578,000060:                                                                                                  #                           INSENSITIVE AND MAY APPEAR IN ANY E-BANK.
003579,000061: 
003580,000062:                                                                                                  #                  SIZE IS THE NUMBER OF REGISTERS INCLUDED BY THE SYMBOL.
003581,000063: 
003582,000064:                                                                                                  #                  N INDICATES THE NATURE OR PERMANENCE OF THE CONTENTS.
003583,000065:                                                                                                  #                       PL  MEANS THAT THE CONTENTS ARE PAD LOADED.
003584,000066:                                                                                                  #                       DSP MEANS THAT THE REGISTER IS USED FOR A DISPLAY.
003585,000067:                                                                                                  #                       PRM MEANS THAT THE REGISTER IS PERMANENT, IE. IT
003586,000068:                                                                                                  #                           IS USED DURING THE ENTIRE MISSION FOR ONE
003587,000069:                                                                                                  #                           PURPOSE AND CANNOT BE SHARED.
003588,000070:                                                                                                  #                       TMP MEANS THAT THE REGISTER IS USED TEMPORARILY OR
003589,000071:                                                                                                  #                           IS A SCRATCH REGISTER FOR THE ROUTINE TO WHICH
003590,000072:                                                                                                  #                           IT IS ASSIGNED.  THAT IS, IT NEED NOT BE SET
003591,000073:                                                                                                  #                           PRIOR TO INVOCATION OF THE ROUTINE NOR DOES IT
003592,000074:                                                                                                  #                           CONTAIN USEFUL OUTPUT TO ANOTHER ROUTINE.  THUS
