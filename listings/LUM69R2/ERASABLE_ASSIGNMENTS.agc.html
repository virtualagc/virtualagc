Source Code

These source-code files are part of a reconstructed copy of LUM69 Revision 2, the flown Apollo 10 Lunar Module (LM) Apollo Guidance Computer (AGC) software. The reconstruction began with source code of Luminary 69 revision 0 previously transcribed from a digitized copy of that program. The code was then updated by back-porting the R-2 lunar potential model from previously-transcribed source code of Luminary 99. These required changes were described in the original Luminary memos 75 and 78. The reconstructed code was verified by matching memory-bank checksums to those listed in drawing 2021152G. Note that page numbers in the reconstructed code match those on the Luminary 069 printout, although the added code would likely have changed page numbers for a real LUM69 Revision 2 listing. Comments from the original source code are prefixed with a single '#' symbol, whereas comments added later are prefixed by "##" or "###". Report any errors noted by creating an issue report at the Virtual AGC project's GitHub repository.

004642,000025:                                                                                                  #           EQUALS IS USED IN TWO WAYS.  IT IS OFTEN USED TO CHAIN A GROUP
004643,000026:                                                                                                  #                  OF ASSIGNMENTS SO THAT THE GROUP MAY BE MOVED WITH THE
004644,000027:                                                                                                  #                  CHANGING OF ONLY ONE CARD.  EXAMPLE.
004645,000028: 
004646,000029:                                                                                                  #                                          X        EQUALS  START
004647,000030:                                                                                                  #                                          Y        EQUALS  X       +SIZE.X
004648,000031:                                                                                                  #                                          Z        EQUALS  Y       +SIZE.Y
004649,000032: 
004650,000033:                                                                                                  #                  (X, Y, AND Z ARE CONSECUTIVE AND BEGIN AT START.       )
004651,000034:                                                                                                  #                  (SIZE.X AND SIZE.Y ARE THE RESPECTIVE SIZES OF X AND Y,
004652,000035:                                                                                                  #                   USUALLY NUMERIC,  IE. 1, 2, 6, 18D ETC.                )
004653,000036: 
004654,000037:                                                                                                  #           EQUALS OFTEN IMPLIES THE SHARING OF REGISTERS (DIFFERENT NAMES
004655,000038:                                                                                                  #                  AND DIFFERENT DATA).  EXAMPLE.
004656,000039: 
004657,000040:                                                                                                  #                                          X        EQUALS Y
004658,000041: 
004659,000042:                                                                                                  #           = MEANS THAT MULTIPLE NAMES HAVE BEEN GIVEN TO THE SAME DATA.
004660,000043:                                                                                                  #                  (THIS IS LOGICAL EQUIVALENCE, NOT SHARING)  EXAMPLE.
004661,000044: 
004662,000045:                                                                                                  #                                          X        =      Y

004664,000047:                                                                                                  #           THE SIZE AND UTILIZATION OF AN ERASABLE ARE OFTEN INCLUDED IN
004665,000048:                                                                                                  #           THE COMMENTS IN THE FOLLOWING FORM.  M(SIZE)N.
004666,000049: 
004667,000050:                                                                                                  #                  M REFERS TO THE MOBILITY OF THE ASSIGNMENT.
004668,000051:                                                                                                  #                       B   MEANS THAT THE SYMBOL IS REFERENCED BY BASIC
004669,000052:                                                                                                  #                           INSTRUCTIONS AND THUS IS E-BANK SENSITIVE.
004670,000053:                                                                                                  #                       I   MEANS THAT THE SYMBOL IS REFERENCED ONLY BY
004671,000054:                                                                                                  #                           INTERPRETIVE INSTRUCTIONS, AND IS THUS E-BANK
004672,000055:                                                                                                  #                           INSENSITIVE AND MAY APPEAR IN ANY E-BANK.
004673,000056: 
004674,000057:                                                                                                  #                  SIZE IS THE NUMBER OF REGISTERS INCLUDED BY THE SYMBOL.
004675,000058: 
004676,000059:                                                                                                  #                  N INDICATES THE NATURE OR PERMANENCE OF THE CONTENTS.
004677,000060:                                                                                                  #                       PL  MEANS THAT THE CONTENTS ARE PAD LOADED.
004678,000061:                                                                                                  #                       DSP MEANS THAT THE REGISTER IS USED FOR A DISPLAY.
004679,000062:                                                                                                  #                       PRM MEANS THAT THE REGISTER IS PERMANENT, IE. IT
004680,000063:                                                                                                  #                           IS USED DURING THE ENTIRE MISSION FOR ONE
004681,000064:                                                                                                  #                           PURPOSE AND CANNOT BE SHARED.
004682,000065:                                                                                                  #                       TMP MEANS THAT THE REGISTER IS USED TEMPORARILY OR
004683,000066:                                                                                                  #                           IS A SCRATCH REGISTER FOR THE ROUTINE TO WHICH
004684,000067:                                                                                                  #                           IT IS ASSIGNED.  THAT IS, IT NEED NOT BE SET
004685,000068:                                                                                                  #                           PRIOR TO INVOCATION OF THE ROUTINE NOR DOES IT
