Source Code

These source-code files were transcribed from scans made from Don Eyles's personal copy of Luminary 069. They were scanned at archive.org's Boston facility, and the scanning was sponsored by Onno Hommes. The code was transcribed from these scans by a team of volunteers who are referenced in the program comments. Comments from the original source code are prefixed with a single '#' symbol, whereas comments added later are prefixed by "##" or "###". In some cases, where similar code blocks exist in previously-transcribed AGC programs (primarily Luminary 99, from Apollo 11) those code blocks were used as a starting point and then corrected to agree with the Luminary 69 scans. The full scans are available at the Virtual AGC project's collection at archive.org, while more-convenient reduced-size (but reduced-quality) images are available at the main Virtual AGC website. Report any errors noted by creating an issue report at the Virtual AGC project's GitHub repository. Notations on the program listing read, in part:

	GAP:  ASSEMBLE REVISION 069 OF AGC PROGRAM LUMINARY BY NASA 2021112-011
	19:02 NOV. 25,1968
Note that the date is the date of the printout, not the date of the program revision.

004671,000032:                                                                                                  #  CONVENTIONS AND NOTATIONS UTILIZED FOR ERASABLE ASSIGNMENTS.
004672,000033: 
004673,000034:                                                                                                  #           EQUALS IS USED IN TWO WAYS.  IT IS OFTEN USED TO CHAIN A GROUP
004674,000035:                                                                                                  #                  OF ASSIGNMENTS SO THAT THE GROUP MAY BE MOVED WITH THE
004675,000036:                                                                                                  #                  CHANGING OF ONLY ONE CARD.  EXAMPLE.
004676,000037: 
004677,000038:                                                                                                  #                                          X        EQUALS  START
004678,000039:                                                                                                  #                                          Y        EQUALS  X       +SIZE.X
004679,000040:                                                                                                  #                                          Z        EQUALS  Y       +SIZE.Y
004680,000041: 
004681,000042:                                                                                                  #                  (X, Y, AND Z ARE CONSECUTIVE AND BEGIN AT START.       )
004682,000043:                                                                                                  #                  (SIZE.X AND SIZE.Y ARE THE RESPECTIVE SIZES OF X AND Y,
004683,000044:                                                                                                  #                   USUALLY NUMERIC,  IE. 1, 2, 6, 18D ETC.                )
004684,000045: 
004685,000046:                                                                                                  #           EQUALS OFTEN IMPLIES THE SHARING OF REGISTERS (DIFFERENT NAMES
004686,000047:                                                                                                  #                  AND DIFFERENT DATA).  EXAMPLE.
004687,000048: 
004688,000049:                                                                                                  #                                          X        EQUALS Y
004689,000050: 
004690,000051:                                                                                                  #           = MEANS THAT MULTIPLE NAMES HAVE BEEN GIVEN TO THE SAME DATA.
004691,000052:                                                                                                  #                  (THIS IS LOGICAL EQUIVALENCE, NOT SHARING)  EXAMPLE.
004692,000053: 
004693,000054:                                                                                                  #                                          X        =      Y
004694,000055: 
004695,000056:                                                                                                  #           THE SIZE AND UTILIZATION OF AN ERASABLE ARE OFTEN INCLUDED IN
004696,000057:                                                                                                  #           THE COMMENTS IN THE FOLLOWING FORM.  M(SIZE)N.
004697,000058: 
004698,000059:                                                                                                  #                  M REFERS TO THE MOBILITY OF THE ASSIGNMENT.
004699,000060:                                                                                                  #                       B   MEANS THAT THE SYMBOL IS REFERENCED BY BASIC
004700,000061:                                                                                                  #                           INSTRUCTIONS AND THUS IS E-BANK SENSITIVE.
004701,000062:                                                                                                  #                       I   MEANS THAT THE SYMBOL IS REFERENCED ONLY BY
004702,000063:                                                                                                  #                           INTERPRETIVE INSTRUCTIONS, AND IS THUS E-BANK
004703,000064:                                                                                                  #                           INSENSITIVE AND MAY APPEAR IN ANY E-BANK.
004704,000065: 
004705,000066:                                                                                                  #                  SIZE IS THE NUMBER OF REGISTERS INCLUDED BY THE SYMBOL.
004706,000067: 
004707,000068:                                                                                                  #                  N INDICATES THE NATURE OR PERMANENCE OF THE CONTENTS.
004708,000069:                                                                                                  #                       PL  MEANS THAT THE CONTENTS ARE PAD LOADED.
004709,000070:                                                                                                  #                       DSP MEANS THAT THE REGISTER IS USED FOR A DISPLAY.
004710,000071:                                                                                                  #                       PRM MEANS THAT THE REGISTER IS PERMANENT, IE. IT
004711,000072:                                                                                                  #                           IS USED DURING THE ENTIRE MISSION FOR ONE
004712,000073:                                                                                                  #                           PURPOSE AND CANNOT BE SHARED.
004713,000074:                                                                                                  #                       TMP MEANS THAT THE REGISTER IS USED TEMPORARILY OR
004714,000075:                                                                                                  #                           IS A SCRATCH REGISTER FOR THE ROUTINE TO WHICH
004715,000076:                                                                                                  #                           IT IS ASSIGNED.  THAT IS, IT NEED NOT BE SET
004716,000077:                                                                                                  #                           PRIOR TO INVOCATION OF THE ROUTINE NOR DOES IT
004717,000078:                                                                                                  #                           CONTAIN USEFUL OUTPUT TO ANOTHER ROUTINE.  THUS
